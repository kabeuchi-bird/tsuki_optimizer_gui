# tsuki_optimize

月配列2-263改変版のかな配列を、コーパス（日本語テキスト）に基づいてタブーサーチで最適化するツールです。CLI版とGUI版があります。

## 概要

- コーパスの文字・バイグラム・トライグラム頻度を集計し、打鍵コストを評価関数として最小化します。
- タブーサーチで文字配置を反復改善します。局所解に陥ると、タブーリスト長を段階的に広げて探索を多様化し、それでも改善しなければ配置をシャッフルして再起動（リスタート）します。
- **GUI版**では最適化の進行をリアルタイムに可視化できます（eframe/egui）。

評価関数の意味は[評価関数](#評価関数)、探索パラメータの詳細は `config.toml` のコメントを参照してください。

### 配列の構造

| レイヤー | 3x10（デフォルト） | 3x11 |
|---------|------------------|------|
| Layer 1 | 30スロット。Dキー（col2）に「、」、Kキー（col7）に「。」固定。゛゜はL1内のみ移動可 | 33スロット。☆（col2）・★（col7）が専用シフトキー、固定文字なし。゛゜はL1内のみ移動可 |
| Layer 2 | 30スロット。D/Kシフト後にアクセス（前置シフト方式） | 33スロット。☆/★シフト後にアクセス（前置シフト方式） |

シフトの方向：左手域のキーは右手側シフトキー（K/★）、右手域のキーは左手側シフトキー（D/☆）で入力します。

## ダウンロード

ビルド済みバイナリは GitHub Releases から直接ダウンロードできます。自分でビルドする必要はありません。

### 手順

1. GitHub のリポジトリページを開く
2. 右サイドバーの **「Releases」** セクションから最新リリースを開く
3. 最新リリースページ下部の **「Assets」** セクションに、OS 別の zip ファイルが添付されている

| ファイル名 | 対象OS |
|-----------|--------|
| `tsuki-optimizer-linux-x86_64.zip` | Linux |
| `tsuki-optimizer-windows-x86_64.zip` | Windows |
| `tsuki-optimizer-macos-arm64.zip` | macOS (Apple Silicon) |

4. 自分の OS に合ったファイルをクリックしてダウンロード
5. zip を展開すると、以下のファイルが入っている

| ファイル | 説明 |
|---------|------|
| `tsuki_optimize` (Windowsは `.exe`) | CLI版（コマンドラインで使う） |
| `gui` (Windowsは `gui.exe`) | GUI版（ウィンドウで操作する） |
| `config.toml` | 設定ファイル（パラメータ調整用） |
| `initial_layout.toml` | ユーザー定義の初期配列（`initial_layout = "user-defined"` 時に使用） |
| `corpus.txt` | サンプルコーパス（自分のテキストに差し替え可） |
| `README.md` | このドキュメント |

6. `gui`（または `gui.exe`）をダブルクリックすれば GUI が起動する

### 自分でビルドする場合

Rust 1.73 以降が必要です。

```bash
cargo build --release
```

ビルド後のバイナリは `target/release/` に生成されます。

#### プラットフォームごとの注意

- **Linux**: GUI のビルドにシステムライブラリが必要です。
  ```bash
  sudo apt-get install -y libxkbcommon-dev libwayland-dev libxcb-render0-dev libxcb-shape0-dev libxcb-xfixes0-dev libfontconfig1-dev
  ```
- **Windows**: 追加依存なし。CLI の `Ctrl+C` / `SIGUSR1` シグナルハンドラは無効になります（GUI の停止ボタンで代替）。
- **macOS**: 追加依存なし。

## 使い方

### CLI モード

```bash
# ビルド後、直接起動（ログは log/ に自動出力される）
./target/release/tsuki_optimize

# CLIオプションで設定を上書き
./target/release/tsuki_optimize --iter 100000 --seed 42

# 3x11キーボードで最適化
./target/release/tsuki_optimize --keyboard-size 3x11

# ログファイルのパスを指定
./target/release/tsuki_optimize --log result.log
```

### GUI モード

```bash
./target/release/gui
```

GUI では以下の機能が利用できます:

- **開始/停止ボタン**: 探索の開始・中断
- **パラメータ入力**: seed、イテレーション数、リスタート閾値、コーパスパス、キーボードサイズ、初期配列モード
- **キーボード表示**: Layer 1 / Layer 2 の配列をリアルタイム更新
- **色分けモード** (3種類):
  - フィットネスマップ: 文字の頻度ランクとスロット難易度ランクのズレを可視化（緑=良い配置、赤=悪い配置）
  - 頻度ヒートマップ: 各キーのコーパス出現頻度（寒色=低頻度、暖色=高頻度）。シフトキーにはL2文字のシフト打鍵頻度を反映
  - 指負荷バランス: 8本の指の負荷を棒グラフで表示。L2文字のシフト打鍵も加算（プリセット使用時は゛゜直前のシフト省略を考慮）
- **スコア推移グラフ**: 現在スコアと最良スコアをリアルタイムプロット。リスタート地点にマーカー表示。ドラッグで移動、ホイールでスクロール、Ctrl+ホイールでズーム、ダブルクリックで自動追従に復帰
- **スコア内訳パネル**: 打鍵数コスト、難易度コスト、バイグラムコスト、トライグラムコストの詳細表示

## 設定ファイル

`config.toml` で全パラメータを設定できます。各項目の詳細はファイル内のコメントを参照してください。省略したフィールドはデフォルト値が使われます。

設定の優先順位（高→低）：CLIオプション → TOMLファイル → デフォルト値

GUI モードでは `config.toml` が存在すれば自動的に読み込まれます。

### 初期配列モード

`run.initial_layout`（または `--initial-layout`）で初期解の作り方を選べます。

| モード | 内容 |
|--------|------|
| `2-263`（デフォルト） | 月配列2-263をベースに、コーパス頻度上位の文字をLayer 1へ寄せた決定的な初期解 |
| `random` | 制約を守りつつ全文字をランダムに配置。`seed` で再現可能 |
| `user-defined` | `initial_layout.toml` に書いた配列を使用。読み込みや検証に失敗するとランダムにフォールバック |

`user-defined` を使う場合は `initial_layout.toml` を編集します（書き方はファイル内のコメント参照）。3x10 と 3x11 の両方を定義でき、使用中のキーボードサイズに対応する方が読み込まれます。

## 評価関数

スコアが小さいほど良い配列です。

1. **打鍵数コスト**（最優先）：Layer 2の文字は2打鍵になるためペナルティ
2. **スロット難易度**：自由に設定可能
3. **バイグラムコスト**：同指連打 > 同手段跨ぎ > 同手 > 交互打鍵 の順に評価。同手連続は小指側（アウトロール）／人差し指側（インロール）のロール方向でボーナス
4. **トライグラムコスト**：準交互打鍵（LLR/RRL等）と、同手3連続のアルペジオ（アウトロール／インロール）にボーナス

各重みは `config.toml` の `[weights]` で調整できます。

## コーパスについて

ひらがな（濁音・半濁音を含む）が含まれていれば、漢字混じりのテキストをそのままコーパスとして利用できます。前処理（漢字除去やかな変換など）は不要です。

### 区切り文字の扱い

配列に含まれない文字（漢字、英数字、記号、空白など）は、n-gram 集計時に **区切り文字** として扱われます。区切り文字の前後で n-gram は連結されません。

例：`「愛してる」と言った` というテキストは、以下のように分解されます。

- 認識されるトークン列：`してる` / `と` / `った`
- 集計されるバイグラム例：`して`、`てる`、`った`
- 集計されるトライグラム例：`して`→`てる`（＝`してる`）、など各トークン内部のみ

そのため、`てると`（トークン境界をまたぐ）や `とった`（`と`と`った`の連結）は n-gram として認識されません。

### 推奨されるコーパス

実際に自分が入力する文章に近いもの（小説、ブログ記事、チャットログ等）を用意するほど、最適化結果が実用的になります。

## ファイル構成

```
tsuki_optimize/
├── Cargo.toml
├── config.toml         # 設定ファイル（パラメータ）
├── initial_layout.toml # ユーザー定義初期配列（initial_layout = "user-defined" 時）
├── .github/workflows/ci.yml   # CI: ビルド・テスト・clippy・成果物アップロード
└── src/
    ├── lib.rs                 # ライブラリルート（各モジュールを pub mod で公開）
    ├── main.rs                # CLI エントリポイント・CLIパース・シグナルハンドラ
    ├── bin/
    │   └── gui/               # GUI（eframe/egui）
    │       ├── main.rs        #   エントリポイント・フォント設定
    │       ├── app.rs         #   App構造体・検索制御ロジック
    │       ├── draw.rs        #   描画メソッド・色分けヘルパー
    │       ├── update.rs      #   eframe::App トレイト実装
    │       └── log_writer.rs  #   ログライター・列挙型定義
    ├── chars.rs               # 文字定義・濁音と半濁音の展開
    ├── layout.rs              # スロット・キーストローク・スワップ操作
    ├── corpus.rs              # コーパス読み込み・n-gram統計
    ├── cost.rs                # 評価関数・差分評価・スコア内訳
    ├── search.rs              # タブーサーチ本体（可動テニュア・デルタキャッシュ・ビットセットタブー判定）
    ├── user_layout.rs         # ユーザー定義初期配列（initial_layout.toml）の読み込み
    └── config.rs              # TOML設定の読み込み
```
